# دليل تشغيل المشروع (خطوة بخطوة)

هذا الملف موجّه لأي شخص هيشغّل المشروع أول مرة، حتى لو ما عندوش خبرة سابقة بـ Laravel. اتبع الخطوات بالترتيب وما تقلق لو حسيت إنها كتيرة - كل خطوة بسيطة لوحدها.

---

## 1. البرامج المطلوبة قبل ما تبدأ

لازم تتثبت على جهازك الأربعة دول:

| البرنامج | الإصدار المطلوب | رابط التحميل |
|---|---|---|
| **PHP** | 8.3 أو أحدث | [windows.php.net/download](https://windows.php.net/download/) |
| **Composer** | أي إصدار حديث | [getcomposer.org](https://getcomposer.org/download/) |
| **Node.js** | أي إصدار LTS حديث (يجيب npm معاه تلقائياً) | [nodejs.org](https://nodejs.org/) |
| **XAMPP** (لتشغيل MySQL) | أي إصدار حديث | [apachefriends.org](https://www.apachefriends.org/) |

### كيف تتأكد إنها متثبتة فعلاً؟

افتح **Command Prompt** أو **PowerShell** (دوّر عليها في قائمة ابدأ) واكتب الأوامر دي وحدة وحدة، بعد كل وحدة اضغط Enter:

```
php -v
composer -v
node -v
npm -v
```

لو كل أمر رجّع لك رقم إصدار (زي `PHP 8.4.23` مثلاً) يبقى تمام. لو ظهرت رسالة زي `'php' is not recognized` يبقى البرنامج ده لسه ما اتثبتش صح أو محتاج تضيفه لـ PATH - أعد تثبيته واتبع تعليمات التثبيت بعناية.

> **ملاحظة لمستخدمي XAMPP:** أحياناً PHP اللي جوا XAMPP يكون إصدار قديم (8.1 أو 8.2) وما يكفيش لهذا المشروع (يحتاج 8.3+). لو عندك نفس المشكلة، ثبّت PHP 8.3+ في مجلد منفصل (مثلاً `C:\php`) وحط مساره في بداية PATH، واستخدم XAMPP فقط لتشغيل MySQL.

### بديل أسهل: Laragon

لو ما عندكش أي حاجة من الجدول أعلاه لسه، أسهل حل إنك تحمّل [Laragon](https://laragon.org/download/) بس - بيجيلك معاه PHP وMySQL وNode.js وComposer كلهم جاهزين مع بعض من غير ما تثبت كل واحد لوحده. لو أصلاً عندك Laragon مثبت، اتبع الملاحظات الخاصة به تحت في كل خطوة (معلّمة بـ 🟢).

> **تأكد من إصدار PHP في Laragon:** كليك يمين على أيقونة Laragon ← **PHP** ← شوف الإصدار المفعّل حالياً. لازم يكون 8.3 أو أحدث. لو الإصدار الموجود أقدم، من نفس القائمة تقدر تختار إصدار تاني لو محمّل، أو تنزّل إصدار أحدث من داخل Laragon نفسه (Tools ← Quick add ← PHP).

## 2. الحصول على المشروع

### الطريقة الأولى: من GitHub

```bash
git clone https://github.com/MGTO21/graduation-project-.git
cd graduation-project-
```

### الطريقة الثانية: من ملف مضغوط (zip)

1. فك ضغط الملف في أي مكان تحب (مثلاً سطح المكتب).
2. افتح Command Prompt أو PowerShell.
3. ادخل لمجلد المشروع بأمر `cd`، مثال:
   ```bash
   cd Desktop\my-project-platform
   ```

من هنا الخطوات نفسها للطريقتين.

> 🟢 **لو بتستخدم Laragon:** الأسهل إنك تحط مجلد المشروع (بعد فك الضغط، أو بعد الـ clone) جوا `C:\laragon\www`. بعدها من نافذة Laragon الرئيسية اضغط زر **Terminal** - هيفتحلك سطر أوامر شغال فعلياً جوا مجلد `www`، ادخل بعدها لمجلد المشروع بـ `cd اسم-المجلد` وكمّل باقي الخطوات منه مباشرة.

## 3. تثبيت المكتبات

داخل مجلد المشروع، شغّل:

```bash
composer install
```

هذا الأمر بيجيب كل مكتبات PHP اللي المشروع محتاجها (موجودة أسماؤها في `composer.json`، بيتم تحميلها في مجلد `vendor/`). ممكن ياخد دقيقة أو دقيقتين حسب سرعة النت.

بعدها:

```bash
npm install
```

نفس الفكرة بس لمكتبات الواجهة (CSS/JS)، بتتحمل في مجلد `node_modules/`.

---

## 4. إعداد ملف البيئة (.env)

المشروع محتاج ملف اسمه `.env` فيه إعدادات قاعدة البيانات وغيرها. انسخ الملف الجاهز:

```bash
copy .env.example .env
```

(على Mac/Linux استخدم `cp .env.example .env` بدلاً منه)

بعدين ولّد مفتاح تشفير فريد للمشروع:

```bash
php artisan key:generate
```

الإعدادات الافتراضية في `.env` مضبوطة لقاعدة بيانات اسمها `elearning_platform` بمستخدم `root` بدون كلمة مرور - وهو بالضبط إعداد XAMPP الافتراضي، فمفيش داعي تغيّر حاجة عادةً. لو قاعدة بياناتك عندها كلمة مرور أو اسم مختلف، عدّل هذي الأسطر في `.env`:

```
DB_DATABASE=elearning_platform
DB_USERNAME=root
DB_PASSWORD=
```

## 5. تشغيل قاعدة البيانات

**لو معاك XAMPP:**
1. افتح **XAMPP Control Panel**.
2. اضغط **Start** بجانب **MySQL** فقط (مش محتاج Apache، المشروع هيتشغل بأمر `artisan serve` بدالو).
3. تأكد إن السطر بجانب MySQL بقى أخضر ومكتوب عليه Running.

**🟢 لو معاك Laragon:**
1. افتح نافذة Laragon الرئيسية.
2. اضغط **Start All** (بيشغّل MySQL وApache مع بعض - عادي حتى لو Apache اشتغل، مش هيعارض المشروع)، أو دوس على MySQL بس لو عايز تشغّله لوحده.
3. تأكد إن الدائرة الصغيرة جنب MySQL بقت خضرا.
4. (اختياري) لو عايز تتصفح قاعدة البيانات بشكل مرئي، كليك يمين على Laragon ← **Database** بيفتحلك HeidiSQL جاهز متصل تلقائياً.

## 6. إنشاء الجداول وتعبئة بيانات تجريبية

```bash
php artisan migrate:fresh --seed
```

هذا الأمر بينشئ كل جداول قاعدة البيانات من الصفر، ويحط بيانات تجريبية جاهزة (حسابات دخول، قسم، مقرر، جدول أسبوعي...) عشان تقدر تجرب المشروع فوراً بدون ما تدخل بيانات يدوياً.

بعدها اربط مجلد التخزين (لازم عشان ملفات المحاضرات المرفوعة تظهر):

```bash
php artisan storage:link
```

---

## 7. بناء ملفات الواجهة (CSS/JS)

```bash
npm run build
```

هذا الأمر بيجمّع ملفات Tailwind CSS والـ JavaScript في نسخة جاهزة للتشغيل (موجودة في `public/build/`). لازم تعمله مرة واحدة على الأقل قبل أول تشغيل، وإلا الصفحات هتفتح بدون تنسيق.

## 8. تشغيل المشروع

```bash
php artisan serve
```

هيظهر لك سطر زي:

```
INFO  Server running on [http://127.0.0.1:8000].
```

افتح الرابط ده في أي متصفح. خلي نافذة الـ Command Prompt هذي مفتوحة طول ما انت شغال على المشروع - لو قفلتها السيرفر هيتوقف.

> 🟢 **ملاحظة لمستخدمي Laragon:** لو حطيت المشروع جوا `www`، Laragon بيعمل تلقائياً رابط جاهز زي `http://اسم-المجلد.test` بدون ما تشغّل `artisan serve` أصلاً. ممكن تستخدمه، لكن الأسهل والأضمن إنك تكمل بـ `php artisan serve` زي فوق - لأن ملف `.env` في المشروع مضبوط على `APP_URL=http://localhost`، ولو استخدمت رابط Laragon التلقائي بدالو ممكن تواجه مشاكل بسيطة في الروابط أو الجلسة (session).

---

## 8.5. تشغيل سيرفر البث المباشر (PeerJS) - مطلوب فقط لو هتجرب البث

لو هتجرب ميزة البث المباشر (WebRTC)، لازم تفتح **نافذة Command Prompt ثالثة** (بجانب نافذة MySQL ونافذة `artisan serve`) وتشغّل فيها:

```bash
npm run peerjs
```

هيظهر لك سطر زي:

```
Started PeerServer on :::, port: 9000, path: /
```

خلي النافذة دي مفتوحة طول فترة تجربة البث. لو قفلتها، صفحات البث (عند المحاضر والطالب) هتفشل تتصل ببعض.

**ليه محتاجينها أصلاً؟** البث المباشر مبني على WebRTC، واللي بيحتاج "سيرفر إشارة" (signaling server) يساعد طرفين (المحاضر والطالب) يلاقوا بعض ويتفقوا على الاتصال المباشر بينهم. جربنا الأول سيرفر إشارة سحابي مجاني عام (بتاع PeerJS نفسها)، لكنه طلع غير مضمون - أحياناً بيرفض الاتصال لأنه سيرفر عام تستخدمه آلاف المشاريع في نفس الوقت وخارج سيطرتنا. فبدّلناه بسيرفر محلي بسيط بيشتغل على جهازك نفسه، وده الأمر اللي بيشغّله.

لو نسيت تشغّله وجربت البث، هتلاقي في صفحة البث رسالة خطأ زي "خطأ في الاتصال: server-error" - ارجع شغّل `npm run peerjs` وأعد تحميل الصفحة.

## 9. تحديث نسخة قديمة عندك (نزّلتها كملف مضغوط قبل كده)

هذا القسم لو أصلاً عندك نسخة شغالة من المشروع نزّلتها قبل كده بـ **Download ZIP** من GitHub (يعني مش عن طريق `git clone`)، وعايز تجيب آخر التحديثات.

بما إن مجلدك الحالي نزل كملف مضغوط، هو أصلاً مش مرتبط بـ git، فـ `git pull` مش هيشتغل عليه مباشرة. أسهل وأضمن طريقة: تجيب نسخة جديدة نظيفة عن طريق git جنب نسختك القديمة، وتنقل بس الحاجات الخاصة بيك (زي ملف `.env` وملفات المحاضرات اللي رفعتها) للنسخة الجديدة.

1. **اعمل نسخة احتياطية من مجلدك الحالي أول حاجة** - غيّر اسمه بدل ما تمسحه، مثلاً من `my-project-platform` إلى `my-project-platform-old`. كده لو احتجت أي حاجة منه تقدر ترجع ليها.

2. اعمل clone للنسخة الجديدة بجانبه (🟢 لو بتستخدم Laragon حطها جوا `C:\laragon\www` زي الأول):
   ```bash
   cd C:\laragon\www
   git clone https://github.com/MGTO21/graduation-project-.git my-project-platform
   cd my-project-platform
   ```

3. **انقل ملف `.env`**: انسخ ملف `.env` من نسختك القديمة (`my-project-platform-old\.env`) للنسخة الجديدة. بعدين قارنه بـ `.env.example` الجديد - لو فيه متغيرات جديدة مش موجودة عندك (مثلاً `APP_LOCALE=ar`)، ضيفها يدوياً لملف `.env` بتاعك.

4. **انقل ملفات المحاضرات المرفوعة** (لو عندك محاضرات رفعتها قبل كده وعايز تحتفظ بيها) من `storage\app\public\lectures` بتاع النسخة القديمة لنفس المسار بالظبط في النسخة الجديدة - دي ملفات فعلية مش موجودة في git أصلاً.

5. ثبّت المكتبات وابني الواجهة زي أي تثبيت عادي (نفس خطوة 3 و7 فوق):
   ```bash
   composer install
   npm install
   npm run build
   ```
   ولو ملف `.env` اللي نقلته ما فيهوش `APP_KEY`، ولّد واحد جديد:
   ```bash
   php artisan key:generate
   ```

6. **قاعدة البيانات - قرار مهم حسب حالتك:**

   - **لو مش لاصق ببياناتك القديمة** (طلاب/مقررات كنت مجرب بيها)، الأسهل تبدأ من نظافة تامة:
     ```bash
     php artisan migrate:fresh --seed
     ```
     هياخد آخر شكل لقاعدة البيانات مع بيانات تجريبية جاهزة من الصفر.

   - **لو عايز تحافظ على البيانات اللي دخلتها بنفسك**، ما تشغّلش `fresh` - شغّل بس:
     ```bash
     php artisan migrate
     ```
     ده بيطبّق المigrations الجديدة فقط (اللي مش موجودة عندك أصلاً) فوق قاعدة بياناتك الحالية من غير ما يمسح حاجة. آخر تحديثين عدّلوا هيكل الجدول شوية (حذف عمود القاعة، وخلّوا القسم في طرح المقرر ينفع يكون فاضي لمقررات مشتركة) - الـ migrations مكتوبة عشان تشتغل صح فوق قاعدة بيانات موجودة زي بتاعتك، فمفروض ما تواجهش مشاكل.

7. اربط التخزين وشغّل السيرفر زي المعتاد:
   ```bash
   php artisan storage:link
   php artisan serve
   ```

8. بعد ما تتأكد إن كل شيء شغال صح على النسخة الجديدة، تقدر تمسح `my-project-platform-old` نهائياً.

### من هنا فصاعداً: تحديث أسهل بكتير

بما إن عندك دلوقتي نسخة حقيقية مرتبطة بـ git، أي تحديث قادم يبقى بسيط جداً - من داخل مجلد المشروع:

```bash
git pull origin main
composer install
npm install
npm run build
php artisan migrate
```

من غير ما تحتاج تكرر خطوات النقل اليدوي (backup / clone جديد / نقل .env) تاني - `git pull` هيجيب التحديثات مباشرة فوق نفس المجلد.

## مشاكل شائعة وحلولها

**"could not find driver" أو خطأ متعلق بـ pdo_mysql**
PHP عندك ناقصه إضافة الاتصال بـ MySQL. افتح ملف `php.ini` (مكانه تقدر تعرفه بأمر `php --ini`) وشيل الـ `;` من أول سطر `extension=pdo_mysql` و`extension=mysqli`، واحفظ وأعد تشغيل الـ terminal.
🟢 في Laragon: كليك يمين على الأيقونة ← **PHP** ← **php.ini** بيفتحلك الملف الصحيح مباشرة بدون ما تدوّر عليه.

**"SQLSTATE[HY000] [2002] ... actively refused"**
معناها MySQL مش شغال. ارجع لخطوة 5 وتأكد إنه Running (في XAMPP Control Panel أو نافذة Laragon حسب اللي معاك).

**الصفحة بتفتح بدون أي تنسيق (تصميم عادي جداً بدون ألوان)**
نسيت خطوة 7 (`npm run build`)، أو شغّلتها قبل ما تنزل المكتبات بـ `npm install`. رجّع الخطوتين بالترتيب.

**"419 Page Expired" لما تحاول تسجل دخول**
امسح الكاش وأعد المحاولة:
```bash
php artisan optimize:clear
```

**فشل رفع ملفات محاضرات كبيرة (فيديو أكبر من كذا ميجا)**
افتح `php.ini` (🟢 في Laragon: كليك يمين ← PHP ← php.ini) وزوّد القيم دي (100 و120 ميجا كافيين لمعظم الحالات):
```
upload_max_filesize = 100M
post_max_size = 120M
```
وأعد تشغيل السيرفر بعدها.

**أيقونة صورة مكسورة بدل شعار الجامعة/الكلية**
طبيعي لو صور الهوية (`public/images/brand/*`) ما كانتش موجودة في النسخة اللي وصلتك - المشروع بيشتغل عادي بدونها.

**صفحة البث بتقول "خطأ في الاتصال: server-error" أو "peer-unavailable"**
نسيت تشغّل سيرفر PeerJS المحلي (خطوة 8.5). شغّل `npm run peerjs` في نافذة منفصلة وسيبها شغالة، وأعد تحميل صفحة البث. لو الخطأ لسه موجود مع إن السيرفر شغال، تأكد إن المحاضر فتح صفحة البث وشاف رسالة "القاعة جاهزة" *قبل* ما الطالب يفتح صفحة المشاهدة (ترتيب فتح الصفحات بيهم).

**رسالة "Could not get an ID from the server" في console المتصفح (F12)**
ده معناه المتصفح ما قدرش يوصل لسيرفر PeerJS المحلي أصلاً - تأكد إن `npm run peerjs` شغال فعلاً (خطوة 8.5) في نافذة منفصلة وما اتقفلتش، وإن البورت `9000` مش محجوز ببرنامج تاني عندك.
